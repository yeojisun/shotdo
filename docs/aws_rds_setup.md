# AWS RDS (MySQL) 100% 무료 프리 티어 생성 가이드

이 가이드는 AWS 프리 티어(Free Tier) 범위 내에서 **단 1원의 요금도 발생하지 않도록** 안전하게 MySQL 데이터베이스를 생성하고 설정하는 최적의 전략을 제공합니다.

---

## 1단계: 기존에 잘못 생성된 DB 삭제하기 (요금 방지 ⚠️)
'손쉬운 생성'으로 만든 DB는 원치 않는 유료 사양(인스턴스 크기, 다중 AZ 등)으로 자동 생성되어 요금이 청구될 수 있으므로, **안전하게 삭제하고 새로 만드는 것이 가장 좋은 전략**입니다.

1. AWS RDS 콘솔의 **[데이터베이스]** 메뉴로 이동합니다.
2. 삭제할 데이터베이스를 선택하고 우측 상단의 **[작업 (Actions)]** -> **[삭제 (Delete)]**를 클릭합니다.
3. ⚠️ **과금 방지 설정**:
   * **최종 스냅샷 생성 (Create final snapshot?)**: **체크 해제 (선택 안 함)** (스냅샷 저장 공간도 오래 두면 요금이 청구됩니다.)
   * **자동 백업 복원 여부**: 체크 해제
   * *"스냅샷을 생성하지 않고... 인스턴스를 영구적으로 삭제할 것임을 인정합니다"* 라는 확인 항목에 **체크**합니다.
4. 입력창에 **`delete me`**(또는 화면에 지시된 문구)를 입력하고 **[삭제]**를 누릅니다. (삭제 완료까지 약 3~5분 소요됩니다.)

---

## 2단계: 100% 무료(Free Tier) 표준 생성 절차 🌟
삭제가 진행되는 동안, 혹은 삭제 완료 후 아래 단계에 맞추어 **[표준 생성]**으로 완전히 안전한 무료 DB를 다시 구축합니다.

1. RDS 콘솔에서 다시 **[데이터베이스 생성]**을 누릅니다.
2. **생성 방식**: **표준 생성 (Standard Create)** 선택
3. **엔진 옵션**: **MySQL** 선택
4. **엔진 버전**: **MySQL 8.0.35** 선택 (가장 안정적이며 마이크로 인스턴스를 지원합니다.)
5. **템플릿 (가장 중요!)**: **프리 티어 (Free Tier)** 선택 
   * *이걸 고르면 유료 옵션들이 기본적으로 비활성화되거나 경고 처리됩니다.*

### 설정 입력
* **DB 인스턴스 식별자**: `shotdo-db`
* **마스터 사용자 이름**: `root`
* **마스터 암호**: 접속 비밀번호 입력 (꼭 메모해 두세요!)

### 인스턴스 구성
* **[버스트 가능 클래스 (t 클래스 포함)]** 선택 ➡️ **`db.t3.micro`** 선택 (100% 무료 대상)

### 스토리지 설정 (요금 방지 ⚠️)
* **할당된 스토리지**: **20 GiB** 입력 (프리 티어 무료 최대 한도)
* ⚠️ **스토리지 자동 조정 활성화 (Enable storage autoscaling)**: **체크 해제**
  * *이걸 해제해야 트래픽이 몰려도 마음대로 용량이 늘어나 결제되는 것을 막을 수 있습니다.*

### 연결 설정 (Vite/Github Pages 외부 접속용)
* **퍼블릭 액세스 가능성 (Publicly accessible)**: **"예 (Yes)"** 선택
* **VPC 보안 그룹**: **새로 생성 (Create new)** 선택
  * **새 VPC 보안 그룹 이름**: `shotdo-db-sg` 입력 (이렇게 하면 방화벽 편집이 매우 쉬워집니다.)

### ⚠️ 추가 구성 (맨 아래 접혀 있는 메뉴 펼치기 - 중요!)
추가 구성(Additional configuration) 버튼을 눌러 메뉴를 펼친 뒤 다음을 설정합니다:
1. **초기 데이터베이스 이름**: **`shotdo`** 입력 (자바 코드가 연결할 이름입니다.)
2. **백업**: **"자동 백업 활성화" 체크 해제** (백업 파일 저장 용량이 초과되어 요금이 청구되는 것을 원천 차단합니다.)
3. **모니터링**: **"향상된 모니터링 활성화" 체크 해제** (모니터링 로그 보관 비용 방지)

## 3단계: 생성 후 딱 한 번만 해야 할 방화벽(보안 그룹) 설정
DB 생성이 끝나면, 외부 컴퓨터에서 접속할 수 있도록 포트를 열어줍니다.

1. DB 상세 페이지의 **[연결 및 보안]** 탭으로 이동합니다.
2. **보안 (Security)** 아래의 **VPC 보안 그룹 (VPC security groups)** 링크(`shotdo-db-sg`)를 클릭합니다.
3. 하단의 **[인바운드 규칙 (Inbound rules)]** -> **[인바운드 규칙 편집]**을 누릅니다.
4. **[규칙 추가]**를 눌러 아래 값을 입력하고 저장합니다:
   * **유형**: **MYSQL/Aurora** (3306 포트 자동 지정)
   * **소스**: **Anywhere-IPv4** (`0.0.0.0/0`)
